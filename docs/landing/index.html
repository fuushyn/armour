Why Armour

Security guardrails with none of the ceremony.

Blocklist-first
Stops bad calls before they leave the IDE.

Pattern matches destructive verbs (rm*, delete*, drop*, sample*) and rejects before tool execution.

Audit trail
Every tool call logged with context.

Who called it, which MCP server, arguments, latency. Exportable JSON for alerts or review.

Zero config
Auto-discovers servers and adds namespaces.

Drop Armour in and keep coding. Namespacing prevents cross-talk between servers.

Policies

Pick a posture, keep shipping.

Strict

Read-only, blocks destructive actions, good for prod branches.

  • Rejects rm*, delete*, drop*
  • No sampling or shell writes
  • Full trace of every call

Moderate

Balanced guardrails for day-to-day work.

  • Blocks obvious destructive commands
  • Allows safe edits and reads
  • Audit log stays on

Permissive

Fast in sandboxes, still logged.

  • No blocking, full visibility
  • Use for experiments and staging
  • Flip to strict for prod
